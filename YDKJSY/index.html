Declaring & Using Variables

  • 'var' --> declares regular/function scope variables
  • 'let' --> declares block scope function {}
  • 'const' --> is like 'let' but must be given a value when declaring & cannot be reassign
  • avoid using 'const' with object value, since those can still be changed eventhough the variable cannot be "re-assigned"
  • 'const' decalred variables are not "unchangeble", they just cannot be "re-assign"
  • 'const often used to assign useful name to a simple primative value

Other syntactic forms that declare identifiers (variables)

  • 'function' --> hello is created in outer scope and references the function
  • 'function' --> myName is created inside function's scope and thus only accessible there
  • 'catch' --> err is block-scoped variable exist only inside 'catch' clause

Functions

  • function delcaration --> the association between the identifider and fuction value happens during comile phase
  • function expression --> not associated with its identifider until that statement during runtime
  • JS treats 'function' as value that can be assigned & passed around
  • functions can receive parameter(variable identifier), which acts as local variable inside function
  • each parameter is assigned the argument value(variable value) that you pass in that position of the call
  • to return more value, can be wrapped into single object/array

Comparisons

  • Equaliy (===) check both the value and the type for primitive data --> structural comparison
  • (===) use identity quality for object values
  • assignment is by reference-copy --> y references the same array as x not another copy of it
  • Coercive (==) a value of one type is converted to its respective representation in other
